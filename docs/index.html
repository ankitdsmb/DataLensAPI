API Implementation Documentation Hub

This document contains the detailed implementation records, technical challenges, and configuration requirements for the successfully deployed APIs within the Forensic API Suite.

1. Instagram Video Scraper - Lite

POST /api/v1/instagram/video-metadata

Use Case: Bulk scraping basic video metadata from Instagram URLs. Simulates mobile traffic to retrieve data without requiring headless browsers.

Example Payload
{
  "urls": ["https://www.instagram.com/reel/XYZ123/"]
}
Example Response
{
  "success": true,
  "data": [
    {
      "url": "https://www.instagram.com/reel/XYZ123/",
      "title": "Funny cat video",
      "likes_count": null,
      "upload_date": "2023-10-25T12:00:00Z"
    }
  ],
  "metadata": {
    "timestamp": "2023-10-25T12:00:05Z",
    "execution_time_ms": 1240
  },
  "error": null
}
Technical Challenges & Workarounds

Instagram aggressively blocks unauthenticated requests from datacenter IPs. The scraper uses got-scraping to spoof TLS fingerprints and mobile headers, which helps slightly. However, full extraction often still requires a valid session cookie, so this 'Lite' version only retrieves high-level og:meta tags or simulates extraction.

Configuration Needed

Requires deployment to a proxy-enabled environment if running at scale to avoid IP bans.

Coding Recall Notes

Used gotScraping.get(url) with mobile headers and regex /<meta property="og:title" content="([^"]+)"/ to extract basic info without executing JavaScript.

2. Link Extractor Pro: URL to HTML List

POST /api/v1/seo/link-extractor

Use Case: Quickly extract all anchor tags (<a>) from a webpage, resolving relative URLs into absolute URLs. Useful for SEO audits and crawling.

Example Payload
{
  "url": "https://example.com/blog",
  "internal_only": true
}
Example Response
{
  "success": true,
  "data": {
    "target": "https://example.com/blog",
    "total_extracted": 12,
    "links": [
      { "href": "https://example.com/about", "text": "About Us", "is_internal": true }
    ]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Dealing with relative URLs (e.g., /contact or ../styles.css) required using the native Node.js new URL(href, baseUrl) to resolve them correctly against the requested root URL.

Configuration Needed

None. Runs perfectly on Vercel's free tier as it only involves simple HTML fetching and parsing.

Coding Recall Notes

Iterated over $('a') with Cheerio. Resolved URLs using new URL($(element).attr('href'), req_url). Filtered external URLs by comparing origin properties.

6. Extract Emails, Contacts, & Socials

POST /api/v1/osint/website-contact-extractor

Use Case: Quickly scan a webpage's HTML to extract email addresses, phone numbers, and social media links using regular expressions.

Example Payload
{
  "url": "https://www.targetcompany.com"
}
Example Response
{
  "success": true,
  "data": {
    "url": "https://www.targetcompany.com",
    "emails": ["contact@targetcompany.com", "support@targetcompany.com"],
    "phones": ["(555) 123-4567"],
    "socials": ["https://twitter.com/targetcompany"]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Regex matching can result in false positives (e.g., matching a random number string as a phone number). The phone regex must be robust enough to handle international formats but strict enough to avoid matching standard numerical IDs. Some websites obfuscate emails using JavaScript (e.g., [at] instead of @), which this fast HTTP scraper won't catch unless run through a headless browser first.

Configuration Needed

None. Runs entirely on Vercel's edge/serverless tier.

Coding Recall Notes

Fetched raw HTML via gotScraping.get(). Used html.match() with predefined Regex patterns for emails, phones, and specific social media domain URLs. Deduplicated results using [...new Set(matches)].
