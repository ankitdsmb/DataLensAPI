1. Instagram Video Scraper - Lite

POST /api/v1/instagram/video-metadata

Use Case: Bulk scraping basic video metadata from Instagram URLs. Simulates mobile traffic to retrieve data without requiring headless browsers.

Example Payload
{
  "urls": ["https://www.instagram.com/reel/XYZ123/"]
}
Example Response
{
  "success": true,
  "data": [
    {
      "url": "https://www.instagram.com/reel/XYZ123/",
      "title": "Funny cat video",
      "likes_count": null,
      "upload_date": "2023-10-25T12:00:00Z"
    }
  ],
  "metadata": {
    "timestamp": "2023-10-25T12:00:05Z",
    "execution_time_ms": 1240
  },
  "error": null
}
Technical Challenges & Workarounds

Instagram aggressively blocks unauthenticated requests from datacenter IPs. The scraper uses got-scraping to spoof TLS fingerprints and mobile headers, which helps slightly. However, full extraction often still requires a valid session cookie, so this 'Lite' version only retrieves high-level og:meta tags or simulates extraction.

Configuration Needed

Requires deployment to a proxy-enabled environment if running at scale to avoid IP bans.

Coding Recall Notes

Used gotScraping.get(url) with mobile headers and regex /<meta property="og:title" content="([^"]+)"/ to extract basic info without executing JavaScript.

2. Link Extractor Pro: URL to HTML List

POST /api/v1/seo/link-extractor

Use Case: Quickly extract all anchor tags (<a>) from a webpage, resolving relative URLs into absolute URLs. Useful for SEO audits and crawling.

Example Payload
{
  "url": "https://example.com/blog",
  "internal_only": true
}
Example Response
{
  "success": true,
  "data": {
    "target": "https://example.com/blog",
    "total_extracted": 12,
    "links": [
      { "href": "https://example.com/about", "text": "About Us", "is_internal": true }
    ]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Dealing with relative URLs (e.g., /contact or ../styles.css) required using the native Node.js new URL(href, baseUrl) to resolve them correctly against the requested root URL.

Configuration Needed

None. Runs perfectly on Vercel's free tier as it only involves simple HTML fetching and parsing.

Coding Recall Notes

Iterated over $('a') with Cheerio. Resolved URLs using new URL($(element).attr('href'), req_url). Filtered external URLs by comparing origin properties.

3. Keyword Density Checker

POST /api/v1/seo/keyword-density

Use Case: Analyze the textual content of a webpage to determine the frequency and percentage density of significant keywords.

Example Payload
{
  "url": "https://example.com/article"
}
Example Response
{
  "success": true,
  "data": {
    "url": "https://example.com/article",
    "analyzed_word_count": 450,
    "keywords": [
      { "keyword": "forensic", "count": 15, "density": "3.33%" },
      { "keyword": "analysis", "count": 10, "density": "2.22%" }
    ]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Cleaning the HTML of noisy elements (scripts, styles, navigation) was essential. Filtering common stop words (like "the", "and") required integrating the external stopword npm package.

Configuration Needed

The stopword module needed to be installed in the workspace dependencies.

Coding Recall Notes

Removed noisy tags via Cheerio $('script, style, nav').remove(). Extracted text via $('body').text(). Tokenized with regex /[^a-zA-Z\s]/g and removed stop words using sw.removeStopwords().

4. Vimeo Profile Scraper

POST /api/v1/vimeo/profile

Use Case: Scrape basic profile information (bio, name, avatar) and extract embedded JSON-LD metadata directly from Vimeo user pages.

Example Payload
{
  "urls": ["https://vimeo.com/user123456"]
}
Example Response
{
  "success": true,
  "data": [
    {
      "url": "https://vimeo.com/user123456",
      "name": "Jane Doe",
      "bio": "Filmmaker based in NY.",
      "avatarUrl": "https://i.vimeocdn.com/...",
      "followers": null,
      "raw_ld_json": { "@context": "http://schema.org", "@type": "Person" }
    }
  ],
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Vimeo uses React, so data like follower counts are not rendered directly in the static HTML. We relied on extracting the <script type="application/ld+json"> block which contains structured SEO data for search engines.

Configuration Needed

None. Standard HTTP request works well.

Coding Recall Notes

Parsed $('script[type="application/ld+json"]').html() using JSON.parse(). Fell back to meta[property="og:title"] tags if JSON-LD parsing failed.

6. Extract Emails, Contacts, & Socials

POST /api/v1/osint/website-contact-extractor

Use Case: Quickly scan a webpage's HTML to extract email addresses, phone numbers, and social media links using regular expressions.

Example Payload
{
  "url": "https://www.targetcompany.com"
}
Example Response
{
  "success": true,
  "data": {
    "url": "https://www.targetcompany.com",
    "emails": ["contact@targetcompany.com", "support@targetcompany.com"],
    "phones": ["(555) 123-4567"],
    "socials": ["https://twitter.com/targetcompany"]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Regex matching can result in false positives (e.g., matching a random number string as a phone number). The phone regex must be robust enough to handle international formats but strict enough to avoid matching standard numerical IDs. Some websites obfuscate emails using JavaScript (e.g., [at] instead of @), which this fast HTTP scraper won't catch unless run through a headless browser first.

Configuration Needed

None. Runs entirely on Vercel's edge/serverless tier.

Coding Recall Notes

Fetched raw HTML via gotScraping.get(). Used html.match() with predefined Regex patterns for emails, phones, and specific social media domain URLs. Deduplicated results using [...new Set(matches)].

7. Czech News Scraper

POST /api/v1/news/czech-scraper

Use Case: Extract article headlines, summaries, and publish dates from popular Czech news websites (like Novinky.cz or iDnes.cz).

Example Payload
{
  "site": "novinky.cz",
  "limit_articles": 5
}
Example Response
{
  "success": true,
  "data": {
    "site": "novinky.cz",
    "total_extracted": 5,
    "articles": [
      {
        "headline": "New Technology Trends",
        "url": "https://www.novinky.cz/...",
        "summary": "A brief overview of the latest trends...",
        "published_at": "Wed, 25 Oct 2023 10:00:00 GMT"
      }
    ]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

HTML structures for news sites change frequently. To ensure reliability without constant maintenance, this scraper targets the site's official RSS/XML feeds rather than parsing the messy HTML homepage.

Configuration Needed

Cheerio must be configured to parse XML correctly.

Coding Recall Notes

Requested the /rss endpoint of the target site. Loaded the response into Cheerio using { xmlMode: true }. Iterated over $('item') to extract title, link, and description nodes.

8. Universal Social Media Scraper

POST /api/v1/osint/universal-social-scraper

Use Case: Provide a single endpoint that detects the social media platform from the URL and extracts high-level profile data (handle, bio, avatar) using generic Open Graph meta tags.

Example Payload
{
  "url": "https://twitter.com/elonmusk"
}
Example Response
{
  "success": true,
  "data": {
    "platform": "twitter",
    "url": "https://twitter.com/elonmusk",
    "title": "Elon Musk (@elonmusk) / X",
    "profile_data": {
      "handle": "elonmusk",
      "bio": "..."
    },
    "avatar_url": "https://pbs.twimg.com/profile_images/..."
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Platforms like Twitter and Instagram aggressively block datacenter IPs and redirect to login pages. Relying solely on og:meta tags works sometimes, but true scraping requires platform-specific undocumented APIs or guest-mode tokens.

Configuration Needed

Proxy rotation highly recommended in production.

Coding Recall Notes

Detected platform using url.includes(). Extracted standard SEO tags using Cheerio: $('meta[property="og:image"]').attr('content'). Used Regex to parse handles directly from the URL path.

9. TikTok Frame Extractor

POST /api/v1/tiktok/frame-extractor

Use Case: Extract high-quality, unwatermarked cover images and animated thumbnails (WebP/GIF) directly from a TikTok video URL.

Example Payload
{
  "video_url": "https://www.tiktok.com/@user/video/123456789"
}
Example Response
{
  "success": true,
  "data": {
    "video_url": "https://www.tiktok.com/@user/video/123456789",
    "title": "Funny dance video",
    "thumbnails": {
      "default": "https://p16-sign.tiktokcdn.com/...",
      "original": "https://p16-sign.tiktokcdn.com/...",
      "animated": "https://p16-sign.tiktokcdn.com/..."
    }
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

TikTok is a heavily obfuscated SPA. The raw video and image links are not in the standard HTML body. They must be extracted from a massive JSON object embedded in a specific script tag used for state hydration.

Configuration Needed

Requires specific desktop/mobile user-agent spoofing to ensure TikTok serves the HTML version rather than forcing an app download interstitial.

Coding Recall Notes

Located the hydration script tag using $('#__UNIVERSAL_DATA_FOR_REHYDRATION__').text(). Parsed the JSON and traversed the object path __DEFAULT_SCOPE__['webapp.video-detail'].itemInfo.itemStruct.video to find the cover URLs.

10. Epicurious Recipes Scraper

POST /api/v1/food/epicurious-scraper

Use Case: Extract structured recipe data including ingredients, instructions, prep time, and nutritional info from Epicurious (or any site using standard Recipe Schema).

Example Payload
{
  "recipe_url": "https://www.epicurious.com/recipes/food/views/pasta"
}
Example Response
{
  "success": true,
  "data": {
    "source_url": "https://www.epicurious.com/recipes/food/views/pasta",
    "name": "Classic Pasta",
    "prep_time": "PT15M",
    "ingredients": ["1 lb pasta", "2 cups tomato sauce"],
    "instructions": ["Boil water", "Add pasta", "Serve with sauce"]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Relying on HTML classes (e.g., .ingredient-list) breaks often because sites redesign frequently. The most resilient method is extracting the hidden JSON-LD structured data (schema.org/Recipe) that search engines rely on.

Configuration Needed

None. Very reliable on serverless architectures.

Coding Recall Notes

Iterated over $('script[type="application/ld+json"]'). Parsed each JSON block looking for an object where @type === 'Recipe'. Mapped the structured schema properties (like recipeIngredient and recipeInstructions) directly to the API response.
