1. Instagram Video Scraper - Lite

POST /api/v1/instagram/video-metadata

Use Case: Bulk scraping basic video metadata from Instagram URLs. Simulates mobile traffic to retrieve data without requiring headless browsers.

Example Payload
{
  "urls": ["https://www.instagram.com/reel/XYZ123/"]
}
Example Response
{
  "success": true,
  "data": [
    {
      "url": "https://www.instagram.com/reel/XYZ123/",
      "title": "Funny cat video",
      "likes_count": null,
      "upload_date": "2023-10-25T12:00:00Z"
    }
  ],
  "metadata": {
    "timestamp": "2023-10-25T12:00:05Z",
    "execution_time_ms": 1240
  },
  "error": null
}
Technical Challenges & Workarounds

Instagram aggressively blocks unauthenticated requests from datacenter IPs. The scraper uses got-scraping to spoof TLS fingerprints and mobile headers, which helps slightly. However, full extraction often still requires a valid session cookie, so this 'Lite' version only retrieves high-level og:meta tags or simulates extraction.

Configuration Needed

Requires deployment to a proxy-enabled environment if running at scale to avoid IP bans.

Coding Recall Notes

Used gotScraping.get(url) with mobile headers and regex /<meta property="og:title" content="([^"]+)"/ to extract basic info without executing JavaScript.

2. Link Extractor Pro: URL to HTML List

POST /api/v1/seo/link-extractor

Use Case: Quickly extract all anchor tags (<a>) from a webpage, resolving relative URLs into absolute URLs. Useful for SEO audits and crawling.

Example Payload
{
  "url": "https://example.com/blog",
  "internal_only": true
}
Example Response
{
  "success": true,
  "data": {
    "target": "https://example.com/blog",
    "total_extracted": 12,
    "links": [
      { "href": "https://example.com/about", "text": "About Us", "is_internal": true }
    ]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Dealing with relative URLs (e.g., /contact or ../styles.css) required using the native Node.js new URL(href, baseUrl) to resolve them correctly against the requested root URL.

Configuration Needed

None. Runs perfectly on Vercel's free tier as it only involves simple HTML fetching and parsing.

Coding Recall Notes

Iterated over $('a') with Cheerio. Resolved URLs using new URL($(element).attr('href'), req_url). Filtered external URLs by comparing origin properties.

3. Keyword Density Checker

POST /api/v1/seo/keyword-density

Use Case: Analyze the textual content of a webpage to determine the frequency and percentage density of significant keywords.

Example Payload
{
  "url": "https://example.com/article"
}
Example Response
{
  "success": true,
  "data": {
    "url": "https://example.com/article",
    "analyzed_word_count": 450,
    "keywords": [
      { "keyword": "forensic", "count": 15, "density": "3.33%" },
      { "keyword": "analysis", "count": 10, "density": "2.22%" }
    ]
  },
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Cleaning the HTML of noisy elements (scripts, styles, navigation) was essential. Filtering common stop words (like "the", "and") required integrating the external stopword npm package.

Configuration Needed

The stopword module needed to be installed in the workspace dependencies.

Coding Recall Notes

Removed noisy tags via Cheerio $('script, style, nav').remove(). Extracted text via $('body').text(). Tokenized with regex /[^a-zA-Z\s]/g and removed stop words using sw.removeStopwords().

4. Vimeo Profile Scraper

POST /api/v1/vimeo/profile

Use Case: Scrape basic profile information (bio, name, avatar) and extract embedded JSON-LD metadata directly from Vimeo user pages.

Example Payload
{
  "urls": ["https://vimeo.com/user123456"]
}
Example Response
{
  "success": true,
  "data": [
    {
      "url": "https://vimeo.com/user123456",
      "name": "Jane Doe",
      "bio": "Filmmaker based in NY.",
      "avatarUrl": "https://i.vimeocdn.com/...",
      "followers": null,
      "raw_ld_json": { "@context": "http://schema.org", "@type": "Person" }
    }
  ],
  "metadata": { ... },
  "error": null
}
Technical Challenges & Workarounds

Vimeo uses React, so data like follower counts are not rendered directly in the static HTML. We relied on extracting the <script type="application/ld+json"> block which contains structured SEO data for search engines.

Configuration Needed

None. Standard HTTP request works well.

Coding Recall Notes

Parsed $('script[type="application/ld+json"]').html() using JSON.parse(). Fell back to meta[property="og:title"] tags if JSON-LD parsing failed.
