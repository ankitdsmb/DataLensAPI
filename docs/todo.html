API Implementation Todo List

This document tracks the detailed implementation plans, technical challenges, and future enhancements for selected forensic scraping tools.

1. Instagram Video Scraper - Lite Medium Difficulty

Implementation Plan

  • Environment: Vercel Serverless Function (Lightweight API).
  • Route: apps/api-gateway/app/api/v1/instagram/video-metadata/route.ts
  • Mechanism: Accept POST request with an array of URLs. Use got-scraping to make HTTP GET requests mimicking a mobile user-agent (e.g., iPhone Safari).
  • Data Extraction: Parse the HTML response to locate the embedded JSON data (often within window.__initialData) or hit Instagram's public GraphQL endpoint. Extract title, likes, comments, and media URLs.
  • Response: Format the extracted data into the standardized JSON envelope.

Technical Challenges

  • Instagram frequently changes the structure of their internal JSON payloads and GraphQL query hashes.
  • Vercel's strict 10-second serverless timeout limit could be exceeded if processing an array of many URLs sequentially.

Hidden Challenges

  • IP Reputation: Standard datacenter IPs (like AWS/Vercel) are heavily monitored and often instantly blocked or redirected to login pages without residential proxies.
  • Session Restrictions: Scraping completely unauthenticated is becoming unreliable.

Feature List Delivered

  • Bulk processing of multiple Instagram URLs.
  • Stealth HTTP requests mimicking real mobile devices.
  • Extraction of: Title/Caption, Likes, Comments, Upload Date, Format, Dimensions.
  • Standardized JSON response wrapper.

Future Enhancement Plan

  • Integrate a residential proxy rotator service.
  • Allow authenticated scraping by accepting a user's sessionid cookie in the request.
  • Implement asynchronous processing with webhooks for massive URL lists to avoid timeouts.
  • Fallback to headless browser strategy if the HTTP API gets completely blocked.

2. Link Extractor Pro: URL to HTML List Easy

Implementation Plan

  • Environment: Vercel Serverless Function (Lightweight API).
  • Route: apps/api-gateway/app/api/v1/seo/link-extractor/route.ts
  • Mechanism: Accept POST request containing a url and an internal_only boolean. Use got-scraping to fetch the HTML content.
  • Data Extraction: Use cheerio to load the HTML and parse all <a> tags. Extract the href attribute and link text.
  • Response: Filter internal links if requested (by comparing domain origin) and return an array of discovered URLs inside the standard JSON envelope.

Technical Challenges

  • Relative vs. Absolute URLs: Many links will be relative (e.g., /about-us). The backend must correctly resolve these against the base URL.
  • Handling malformed HTML or single-page applications (SPAs) that load links dynamically via JavaScript after the initial HTML load.

Hidden Challenges

  • Some target websites block automated requests entirely (e.g., Cloudflare 403 Forbidden).

Feature List Delivered

  • Extracts all URLs and anchor text from a target webpage.
  • Filters between internal and external links.
  • Automatically resolves relative URLs to absolute URLs.
  • Returns data in a clean, structured JSON format.

Future Enhancement Plan

  • Add crawling depth capabilities (e.g., extracting links from the target page, and then links from those pages).
  • Provide a CSV download option for SEO analysts.
  • Integrate a headless browser fallback (Playwright on Render) for scraping links from React/Vue SPAs that don't server-side render.

3. Keyword Density Checker Easy

Implementation Plan

  • Environment: Vercel Serverless Function (Lightweight API).
  • Route: apps/api-gateway/app/api/v1/seo/keyword-density/route.ts
  • Mechanism: Accept POST request with a target URL. Use got-scraping to fetch the HTML.
  • Data Extraction: Use cheerio to extract visible text (ignoring <script>, <style>, <nav>, <footer>). Use a lightweight NLP tokenizer or regex to split text into words, filter out common stop words (e.g., "the", "and", "is"), and calculate the frequency and percentage density of each keyword.
  • Response: Return a sorted list of keywords by density in the standardized JSON wrapper.

Technical Challenges

  • Accurately extracting *only* the main content text. Headers, footers, sidebars, and navigation menus can heavily skew the keyword density calculations.
  • Language Support: Filtering out stop words requires a predefined list of words per language. A robust implementation needs to detect the language or accept it as a parameter.

Hidden Challenges

  • Multi-word phrases (n-grams): Simply counting single words is often less valuable than counting 2-word or 3-word phrases (e.g., "search engine optimization" vs "search").

Feature List Delivered

  • Fetches HTML content from a given URL.
  • Cleans HTML to extract only the readable text body.
  • Calculates the total word count, keyword frequency, and keyword density percentages.
  • Removes basic English stop words.

Future Enhancement Plan

  • Implement n-gram (bigram/trigram) analysis for multi-word phrases.
  • Support multiple languages (e.g., French, Spanish, German) for stop word filtering.
  • Use the Readability.js library (like Firefox Reader View) to better isolate the main article content before analyzing.

4. Vimeo Profile Scraper Easy

Implementation Plan

  • Environment: Vercel Serverless Function (Lightweight API).
  • Route: apps/api-gateway/app/api/v1/vimeo/profile/route.ts
  • Mechanism: Accept POST request containing an array of Vimeo profile urls. Use got-scraping to fetch the HTML.
  • Data Extraction: Parse the HTML using cheerio. Search for a <script type="application/ld+json"> block containing JSON-LD structured data. This object often contains the profile name, bio, follower counts, and high-resolution profile image URL.
  • Response: Wrap the extracted JSON-LD object (or selected fields) into the standardized JSON envelope.

Technical Challenges

  • Vimeo heavily uses React for rendering. The JSON-LD schema may not always contain all the data we want (e.g., total likes across all videos). We might also need to find a vimeo.config global JavaScript object embedded in the page source and parse its JSON string.
  • Handling multiple profiles in an array sequentially might hit Vercel's 10-second serverless timeout.

Hidden Challenges

  • Rate Limiting: Standard datacenter IPs will eventually get temporarily blocked (429 Too Many Requests) by Vimeo if requests are too frequent.

Feature List Delivered

  • Fetches HTML from a Vimeo user profile URL.
  • Extracts the raw JSON-LD metadata and/or internal config objects.
  • Extracts Profile Name, Bio, HD Avatar Image URL, Follower count, Following count.
  • Standardized JSON response wrapper.

Future Enhancement Plan

  • Scrape the list of recent videos (thumbnails, titles, views) directly from the profile page.
  • Implement proxy rotation to avoid rate limits.
  • Use the official Vimeo API if possible, bypassing the need to scrape HTML entirely (requires an API token).

5. 9GAG Video Downloader – HD Quality Easy

Implementation Plan

  • Environment: Vercel Serverless Function (Lightweight API).
  • Route: apps/api-gateway/app/api/v1/9gag/downloader/route.ts
  • Mechanism: Accept POST request with a 9GAG post_url. Use got-scraping to fetch the HTML content.
  • Data Extraction: Parse the HTML using cheerio. Search for a <script type="application/ld+json"> block containing the video's title and metadata, or locate the primary <video> tag and extract the src attribute (usually a .mp4 or .webm file URL hosted on 9GAG's CDN). Also, try extracting the upvotes/comments count from other DOM elements if available in the initial load.
  • Response: Return the video title, the direct MP4 URL, and the upvotes/comments count in the standardized JSON wrapper.

Technical Challenges

  • Finding the direct `.mp4` URL if 9GAG serves the video via a blob URL or HLS stream (unlikely for simple posts, but possible for longer content).
  • Extracting the exact metadata (upvotes) might be difficult if 9GAG loads this data asynchronously via a separate API call *after* the initial HTML loads.

Hidden Challenges

  • 9GAG may employ anti-bot measures (like Cloudflare) that block standard HTTP requests from datacenter IPs.

Feature List Delivered

  • Accepts a 9GAG post URL.
  • Fetches the raw HTML content.
  • Extracts the direct HD `.mp4` video URL from the `
  • Extracts the post title and (if possible) the upvotes/comments count.
  • Returns a structured JSON response.

Future Enhancement Plan

  • Implement an audio extraction feature (downloading the `.mp4` and converting it to `.mp3` using FFmpeg). This would likely require moving the API to the Render (heavyweight) service since Vercel serverless functions have limited disk space and CPU for FFmpeg processing.
  • Provide direct download links (proxying the CDN file) rather than just returning the CDN URL. This requires handling potentially large file sizes and bandwidth.
  • Add support for extracting image galleries (multiple images) from a single 9GAG post URL.
